Security Model

problem

The Challenge

Traditional perimeter-based security models assume everything inside a network is trustworthy — a flawed assumption in the era of remote work, cloud computing, and sophisticated cyber threats. Once an attacker gets past the perimeter, they often have unrestricted access to critical systems. The Zero Trust model addresses this by enforcing strict, continuous verification for every entity.

approach

How It Was Built

  • Designed the complete Zero Trust architecture for a simulated enterprise environment
  • Implemented webcam-based biometric enrollment and login verification
  • Built JWT-protected API endpoints with role-based access control
  • Developed an admin dashboard for real-time security policy management
  • Integrated attendance tracking and file access APIs with detailed logging
how it works

System Working

Users register with email credentials and simultaneously enroll their webcam image as a biometric factor. At login, the system verifies both factors — a password check and a live webcam match against the stored hash. On success, a short-lived JWT is issued. Every subsequent API call validates that token, and the user's role (Employee, Manager, or Admin) determines which resources they can access. All actions are logged to an audit trail.

challenges

Challenges Faced

  • Designing a secure and scalable architecture that remains performant under concurrent access
  • Implementing robust image hash matching for biometric verification without a GPU pipeline
  • Preventing token replay attacks and enforcing session expiry correctly
roadmap
