Mini Compiler

Problem Statement

The project involves designing and implementing a simple compiler for a C language. The compiler should be able to parse source code, generate intermediate code, and produce executable output. The goal is to demonstrate the fundamental concepts of compiler design, including lexical analysis, syntax analysis, semantic analysis, and code generation.

Purpose of the Project

The Mini Compiler project aims to provide a hands-on understanding of the compiler construction process. By building a simple compiler, students can grasp the intricacies of translating high-level code into machine-readable instructions. This project serves as an educational tool to illustrate the principles of lexical analysis, syntax analysis, semantic analysis, and code generation.

Approach

  • Designed the architecture for the Mini Compiler implementation.
  • Developed a simulated environment to demonstrate compiler concepts.
  • Implemented lexical, syntax, and semantic analysis components.
  • Created a user-friendly interface for managing and monitoring security policies.
  • Integrated logging and reporting features to track security events.

Compiler Workflow

  • Source Code Input
  • Lexical Analysis
  • Token Generation
  • Syntax Analysis
  • Symbol Table Management
  • Intermediate Code Generation
  • Error Reporting

Working

The Mini Compiler processes source code through multiple phases. First, the lexical analyzer scans the input and identifies tokens such as keywords, identifiers, operators, and constants. The parser then checks whether the code follows the defined grammar rules. A symbol table is maintained to store identifiers and their attributes. Finally, intermediate code is generated and stored for further processing. Any syntax errors encountered during compilation are reported to the user.
