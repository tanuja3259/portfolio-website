problem

The Challenge

Compilers are one of the most complex pieces of software in existence — translating human-readable source code into machine instructions through multiple precise phases. This project aims to build a stripped-down but fully functional compiler for a subset of C, demonstrating each phase from tokenization through to intermediate code generation, to deepen understanding of how modern compilers work under the hood.

approach

How It Was Built

  • Defined a formal grammar for a subset of the C language
  • Built a hand-written lexer that identifies tokens, keywords, operators, and literals
  • Implemented a recursive descent parser to validate grammar rules
  • Designed a symbol table to track identifiers, types, and scopes
  • Generated three-address intermediate code for further optimization
  • Added structured error messages with line numbers and context
how it works

Compiler Workflow

The source file is read character-by-character by the lexer, which emits a stream of classified tokens. The parser consumes that stream, validating it against the grammar using recursive descent. As declarations are encountered, the symbol table is updated. Once parsing succeeds, the code generator walks the syntax tree to emit three-address instructions. Any violation at any stage produces a descriptive error pointing back to the offending line.

challenges

Challenges Faced

  • Designing unambiguous grammar rules that cover enough C constructs to be useful
  • Handling operator precedence and associativity correctly in the parser
  • Managing nested scopes in the symbol table without memory leaks
  • Generating meaningful, actionable error messages for malformed input
roadmap

Future Enhancements

  • Full semantic analysis — type checking, scope resolution
  • Code optimization passes (constant folding, dead code elimination)
  • x86 assembly code generation backend
  • Support for pointers, structs, and function prototypes
  • Graphical visualization of the parse tree
