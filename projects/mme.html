Multimedia Encryption
& Decryption

A full-stack MERN web application for securely encrypting and decrypting images, audio, video, and documents — with JWT authentication, SMTP email verification, and a clean, intuitive interface.

View on GitHub Back to Portfolio
problem

The Challenge

As multimedia files — images, audio, and videos — are increasingly exchanged over the internet, protecting sensitive content from unauthorized access has become critical. Traditional file-sharing methods expose data to significant security risks, and most existing tools are either too complex for everyday users or sacrifice security for convenience. This project bridges that gap with a web-based, user-friendly encryption platform.

approach

How It Was Built

  • Designed a secure REST API in Express.js for file upload, encryption, and decryption
  • Built a React.js frontend with drag-and-drop file upload and real-time feedback
  • Implemented strong symmetric encryption for all supported file types
  • Integrated JWT-based authentication and SMTP email verification
  • Used MongoDB to manage user accounts, file metadata, and encryption keys
  • Ensured efficient streaming for large files to avoid memory bottlenecks
how it works

System Working

After registering and verifying their email, users log in with a JWT-issued session. They upload a file through the interface and choose to encrypt or decrypt it. The backend processes the file using the chosen encryption algorithm and returns a secured or restored output. For decryption, the correct secret key is required — without it, the content remains inaccessible. All file operations are logged and tied to the authenticated user.

challenges

Challenges Faced

  • Handling multiple binary file formats (images, audio, video, PDF) without corruption
  • Streaming large files through the encryption pipeline efficiently
  • Keeping encryption keys secure while keeping the UX simple
  • Ensuring data integrity — that decrypted output exactly matches the original
roadmap

Future Enhancements

  • Advanced encryption modes (AES-GCM, ChaCha20-Poly1305)
  • Multi-user secure file sharing with per-recipient keys
  • Real-time collaborative encryption sessions
  • Security monitoring and anomaly alerting dashboard
